# Data Preprocessing

## 개요

이 문서는 AI Materials Discovery Platform의 데이터 전처리 구조를 설명한다.

현재 전처리는 크게 2단계로 나뉜다.

1. 1차 전처리
   - 도메인 기준 검증
   - 형식 오류 처리
   - 결측치 처리
   - 이상치 처리
2. 2차 전처리
   - 합금 지표(feature engineering) 생성

구현 위치:

- `src/engine/data_engine.py`
- `src/gui/main_window.py`

## 전처리 흐름

전처리는 `DataEngine.load_data()` 실행 시 아래 순서로 진행된다.

1. Excel 파일을 읽는다.
2. 원본 조성/공정/타깃 컬럼을 숫자형으로 변환한다.
3. 잘못된 형식 값을 집계한다.
4. 결측치 처리 전략을 적용한다.
5. 도메인 기준과 IQR 기준으로 이상치를 감지한다.
6. 이상치를 보정하거나 제거한다.
7. 전처리 결과 인덱스를 다시 정렬한다.
8. 2차 전처리가 켜져 있으면 합금 지표를 추가 생성한다.
9. 결과를 `self.df`와 `last_quality_report`에 저장한다.

## 1차 전처리

### 1. 도메인 기준 검증

도메인 검증은 단순 통계 이상치만 보는 것이 아니라, 재료공학적으로 말이 되는 범위인지 함께 확인한다.

현재 기준은 2개 부류로 나뉜다.

- 오스테나이트 조성 기준
- 고온 특성 기준

기본 범위 예시:

### 오스테나이트 조성 기준

- `Cr`: `16 ~ 26`
- `Ni`: `3.5 ~ 37`
- `Mo`: `0 ~ 5`
- `Mn`: `0 ~ 9`
- `Cu`: `0 ~ 4`
- `N`: `0 ~ 0.4`
- `C`: `0 ~ 0.15`

### 고온 특성 기준

- `Solution_treatment_temperature`: `900 ~ 1500`
- `Temperature (K)`: `273 ~ 1422`
- `Water_Quenched_after_s.t.`: `0 ~ 1`
- `Air_Quenched_after_s.t.`: `0 ~ 1`

도메인 범위는 UI의 `도메인 기준 설정`에서 직접 수정할 수 있다.

### 2. 형식 오류 처리

수치형 컬럼은 모두 `pd.to_numeric(..., errors="coerce")`를 통해 숫자로 변환한다.

지원 방식:

- `잘못된 값을 NaN으로 변환`
- `잘못된 값이 있는 행 제거`

형식 오류 개수는 `invalid_type_cells`로 기록된다.

### 3. 결측치 처리

지원 방식:

- `평균값으로 채우기`
- `중앙값으로 채우기`
- `주변 값으로 예측(KNN)`
- `해당 행 제거`

권장 기본값:

- `중앙값으로 채우기`

### 4. 이상치 처리

이상치는 다음 두 기준을 함께 사용한다.

- 도메인 기준 범위
- IQR 기반 통계 범위

지원 방식:

- `감지 범위로 보정`
- `이상치 행 제거`
- `표시만 하고 유지`

권장 기본값:

- `감지 범위로 보정`

### 5. 이상치 민감도

IQR 기반 감지 강도를 조절하는 값이다.

- 기본값: `1.5`
- 더 엄격하게 보고 싶으면: `1.0 ~ 1.3`
- 더 느슨하게 보고 싶으면: `2.0 이상`

## 2차 전처리

2차 전처리는 1차 전처리로 정리된 데이터를 바탕으로 합금 지표를 추가 생성하는 단계다.

UI에서는 `합금 지표 생성 전처리` 체크박스로 켜고 끌 수 있다.

기본값은 `사용`이다.

### 생성되는 합금 지표

- `Cr_Ni_ratio`
- `C_plus_N`
- `Ni_eq`
- `Cr_eq`

### 계산식

- `Cr_Ni_ratio = Cr / Ni`
- `C_plus_N = C + N`
- `Ni_eq = Ni + 30*C + 0.5*Mn`
- `Cr_eq = Cr + Mo + 1.5*Si + 0.5*Nb`

### 왜 2차 전처리로 분리하는가

이 단계는 데이터 정제가 아니라 feature engineering에 가깝다.

즉 역할이 다르다.

- 1차 전처리: 데이터가 정상인지 정리
- 2차 전처리: 재료공학적으로 더 의미 있는 입력 변수 생성

따라서 순서는 아래가 가장 적절하다.

1. 도메인 검증 및 품질 처리
2. 합금 지표 생성
3. 스케일링
4. 모델 학습

## 모델 학습과의 연결

전처리 완료 후 `preprocess_data()`에서 학습용 입력과 타깃을 분리한다.

이때 입력 변수는 다음 순서로 사용된다.

1. 원본 조성/공정 변수
2. 2차 전처리로 생성된 합금 지표

그 다음 `StandardScaler`로 입력과 타깃을 각각 스케일링한다.

즉, 합금 지표 생성은 모델 내부가 아니라 전처리 단계에서 먼저 수행된다.

## 전처리 결과 확인

UI의 `데이터 전처리` 탭에서는 전처리 완료 후 결과를 표 전체로 확인할 수 있다.

특징:

- 전처리 완료 데이터를 전체 행 기준으로 출력
- 전처리 후 순서대로 다시 인덱스를 정렬
- 설정이 바뀌면 이전 결과는 무효화
- `전처리 결과 초기화` 버튼으로 결과 상태를 리셋 가능

## 전처리 리포트

전처리 후 `DataEngine.last_quality_report`에는 아래 정보가 저장된다.

- `rows_before`
- `rows_after`
- `invalid_type_cells`
- `missing_cells_before`
- `missing_cells_after`
- `outlier_cells`
- `domain_range_cells`
- `rows_removed_by_missing`
- `rows_removed_by_outlier`
- `missing_strategy`
- `outlier_strategy`
- `invalid_type_strategy`
- `feature_engineering_enabled`
- `engineered_features_added`
